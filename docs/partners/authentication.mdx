All requests to the Partner API must be authenticated with a secret **API key**
issued by the Eïa team.

## API keys

An API key is a long-lived secret that:

- Identifies your organisation (the partner).
- Can be scoped to a single venue, or left platform-wide.
- Carries a set of [scopes](#scopes) that gate which endpoints it can call.
- Can be revoked at any time.

Keys are generated and stored securely on Eïa's side (the secret is held in an
encrypted vault and only its prefix is kept in clear for lookup). The Eïa team
provisions your key and shares it with you once.

<Warning>
  Treat API keys like passwords. Never expose them in client-side code
  (browsers, mobile apps). All calls to the Partner API must originate from
  your server.
</Warning>

## Using your key

Send the key on every request, using **either** of these headers:

```http
Authorization: Bearer sk_live_ab12cd34_0f9e8d7c6b5a4938271605f4e3d2c1b0a9f8e7d6
```

```http
X-Api-Key: sk_live_ab12cd34_0f9e8d7c6b5a4938271605f4e3d2c1b0a9f8e7d6
```

If both are present, `Authorization: Bearer` takes precedence.

### Example

<CodeGroup>

```bash curl
curl https://api.saoma.io/v1/venues \
  -H "Authorization: Bearer $EIA_API_KEY"
```

```javascript Node.js
const res = await fetch("https://api.saoma.io/v1/venues", {
  headers: {
    Authorization: `Bearer ${process.env.EIA_API_KEY}`,
  },
});
const { data } = await res.json();
```

```python Python
import os, requests

res = requests.get(
    "https://api.saoma.io/v1/venues",
    headers={"Authorization": f"Bearer {os.environ['EIA_API_KEY']}"},
)
data = res.json()["data"]
```

</CodeGroup>

## Key format

API keys follow the pattern `sk_live_{id}_{secret}` where:

- `sk` stands for _secret key_, `live` for the production environment.
- `id` is a short hex identifier (used for lookup and display).
- `secret` is a long hex random string — the part you must keep confidential.

## Rotation and revocation

To rotate a key:

1. Request a new key from your Eïa contact.
2. Deploy the new key to your servers.
3. Confirm traffic is flowing on the new key.
4. Ask the Eïa team to revoke the old key.

A revoked key is rejected immediately with `401`
`{ "error": "Invalid or revoked API key" }`.

## Scopes

Each API key is tagged with one or more scopes. Calls to an endpoint whose
scope your key lacks return `403` with
`{ "error": "Insufficient scope", "missing": ["…"] }`.

| Scope             | Status      | Grants                                                  |
| ----------------- | ----------- | ------------------------------------------------------- |
| `venues:read`     | Available   | `GET /v1/venues`, `GET /v1/venues/{id}`                 |
| `treatments:read` | Available   | `GET /v1/venues/{id}/treatments`                        |
| `availability:read` | Coming soon | Real-time availability endpoint (not yet released)    |
| `bookings:write`  | Coming soon | Create / cancel bookings (not yet released)             |
