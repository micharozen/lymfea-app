---
title: "Get treatment availability"
api: "GET /v1/venues/{venue_id}/availability/treatments/{treatment_id}"
description: "Real-time bookable slots for a specific treatment of a venue."
---

Same as [Get venue availability](/api-reference/availability/get-availability),
but restricted to a single treatment. Slots are computed with that treatment's
duration and lead time, and only therapists qualified for it are considered, so
the result is a subset of the whole-venue availability. Requires the
`availability:read` scope.

## Path parameters

<ParamField path="venue_id" type="string" required>
  The venue id or slug.
</ParamField>

<ParamField path="treatment_id" type="string" required>
  The treatment id. Must belong to the venue.
</ParamField>

## Query parameters

<ParamField query="variant_id" type="string">
  Restrict the duration to a specific [variant](/concepts/treatments). Defaults
  to the treatment's default variant.
</ParamField>

<ParamField query="date" type="string (ISO date)">
  A single day, `YYYY-MM-DD`. Mutually exclusive with `from`/`to`.
</ParamField>

<ParamField query="from" type="string (ISO date)">
  Start of the range, inclusive. Requires `to`.
</ParamField>

<ParamField query="to" type="string (ISO date)">
  End of the range, inclusive. Max `from + 62 days`.
</ParamField>

You must provide **either** `date` **or** both `from` and `to`.

## Response

Identical shape to [Get venue availability](/api-reference/availability/get-availability):
`{ data: { date, slots } }` in single-day mode, `{ data: { from, to, days } }`
in range mode.

## Errors

| HTTP | Body                                  | Meaning                                  |
| ---- | ------------------------------------- | ---------------------------------------- |
| 400  | `{ "error": "Provide \`date\` or \`from\`/\`to\`" }` | Missing or conflicting period params |
| 403  | `{ "error": "Forbidden" }`            | Your key is scoped to another venue/org  |
| 404  | `{ "error": "Venue not found" }`      | Unknown venue                            |
| 404  | `{ "error": "Treatment not found" }`  | Treatment does not belong to the venue   |
| 404  | `{ "error": "Variant not found" }`    | `variant_id` does not belong to the treatment |

## Example

<RequestExample>

```bash curl
curl "https://api.saoma.io/v1/venues/lutetia-paris/availability/treatments/a1c2e3f4-5678-49ab-cdef-0123456789ab?date=2026-06-20" \
  -H "Authorization: Bearer $SAOMA_API_KEY"
```

</RequestExample>

<ResponseExample>

```json 200
  "data": {

    "out_of_hours_surcharge_percent": 20,
    "slots": [

      { "time": "09:30", "out_of_hours": false, "available_capacity": 1 },
      { "time": "14:00", "out_of_hours": false, "available_capacity": 2 }

  }
}
```

</ResponseExample>
