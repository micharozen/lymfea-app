---
title: "Get venue availability"
api: "GET /v1/venues/{venue_id}/availability"
description: "Real-time bookable slots for a whole venue."
---

Returns the currently bookable slots for a venue. Requires the
`availability:read` scope.

Two query modes (mutually exclusive):

- **Single day** — pass `date` to get every free slot of that day.
- **Date range** — pass `from` and `to` to get the days that have at least one
  free slot (useful to paint a month calendar). Range is capped at **62 days**.

<Tip>
  Results reflect live availability at query time. A slot returned here may be
  taken by another customer before a booking is made. Treat availability as a
  hint, not a reservation.
</Tip>

## Path parameters

<ParamField path="venue_id" type="string" required>
  The venue id or slug.
</ParamField>

## Query parameters

<ParamField query="date" type="string (ISO date)">
  A single day, `YYYY-MM-DD`. Mutually exclusive with `from`/`to`.
</ParamField>

<ParamField query="from" type="string (ISO date)">
  Start of the range, inclusive, `YYYY-MM-DD`. Requires `to`.
</ParamField>

<ParamField query="to" type="string (ISO date)">
  End of the range, inclusive, `YYYY-MM-DD`. Max `from + 62 days`.
</ParamField>

You must provide **either** `date` **or** both `from` and `to`.

## Response

In **single-day** mode:

<ResponseField name="data.date" type="string (ISO date)">
  The requested day.
</ResponseField>

<ResponseField name="data.slots" type="string[]">
  Bookable start times for that day, in the venue's local time, format `HH:MM`,
  aligned on the venue's `slot_interval`. See [Availability](/concepts/availability).
</ResponseField>

In **range** mode:

<ResponseField name="data.from" type="string (ISO date)" />
<ResponseField name="data.to" type="string (ISO date)" />

<ResponseField name="data.days" type="string[]">
  The dates in the range (venue-local) that have at least one bookable slot.
  Fully-booked or closed days are omitted.
</ResponseField>

## Errors

| HTTP | Body                                         | Meaning                                  |
| ---- | -------------------------------------------- | ---------------------------------------- |
| 400  | `{ "error": "Provide \`date\` or \`from\`/\`to\`" }` | Missing or conflicting period params     |
| 400  | `{ "error": "Range must not exceed 62 days" }` | Range too wide / inverted              |
| 403  | `{ "error": "Forbidden" }`                   | Your key is scoped to another venue/org  |
| 404  | `{ "error": "Venue not found" }`             | Unknown venue                            |

## Example

<RequestExample>

```bash Single day
curl "https://api.saoma.io/v1/venues/lutetia-paris/availability?date=2026-06-20" \
  -H "Authorization: Bearer $SAOMA_API_KEY"
```

```bash Range
curl "https://api.saoma.io/v1/venues/lutetia-paris/availability?from=2026-06-20&to=2026-06-30" \
  -H "Authorization: Bearer $SAOMA_API_KEY"
```

</RequestExample>

<ResponseExample>

```json Single day
{
  "data": {
    "date": "2026-06-20",
    "slots": ["09:00", "09:30", "10:00", "14:00", "14:30"]
  }
}
```

```json Range
{
  "data": {
    "from": "2026-06-20",
    "to": "2026-06-30",
    "days": ["2026-06-20", "2026-06-21", "2026-06-24"]
  }
}
```

</ResponseExample>
