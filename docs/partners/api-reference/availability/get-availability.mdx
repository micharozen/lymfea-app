---
title: "Get availability"
api: "GET /v1/availability"
description: "Fetch real-time bookable slots for one or more venues."
---

The central endpoint for partners. Returns the slots that are currently
bookable for a venue (or a batch of venues), over a date range of up to
31 days.

<Tip>
  Results reflect live availability at query time. A slot returned here may
  still be taken by another customer before you call `POST /v1/bookings`.
  Handle `SLOT_TAKEN` (409) gracefully on the booking call.
</Tip>

## Query parameters

<ParamField query="venue_id" type="string">
  A single venue id or slug. Mutually exclusive with `venue_ids`.
</ParamField>

<ParamField query="venue_ids" type="string[]">
  Comma-separated list of venue ids (max 20). Use this to fan-out a single
  availability query across a marketplace page.
</ParamField>

<ParamField query="treatment_id" type="string">
  Restrict to a specific treatment. If omitted, slots are returned for every
  bookable treatment of the venue(s).
</ParamField>

<ParamField query="variant_id" type="string">
  Restrict to a specific variant. Implies `treatment_id`.
</ParamField>

<ParamField query="from" type="string (ISO date)" required>
  Start of the date range, inclusive. Format `YYYY-MM-DD`.
</ParamField>

<ParamField query="to" type="string (ISO date)" required>
  End of the date range, inclusive. Max `from + 30 days`.
</ParamField>

<ParamField query="guests" type="integer" default="1">
  Number of guests. Used to filter out variants whose `guest_count` does not
  match.
</ParamField>

## Response

<ResponseField name="data" type="VenueAvailability[]">
  One entry per venue queried.
</ResponseField>

<ResponseField name="data[].venue_id" type="string" />

<ResponseField name="data[].timezone" type="string">
  Reminder of the venue's timezone, for display.
</ResponseField>

<ResponseField name="data[].days" type="DayAvailability[]">
  One entry per date in the requested range that has at least one available
  slot. Dates with no availability are omitted.
</ResponseField>

<ResponseField name="data[].days[].date" type="string (ISO date)">
  Date in the venue's local timezone.
</ResponseField>

<ResponseField name="data[].days[].slots" type="Slot[]">
  Array of [Slot](/concepts/availability) objects for that day.
</ResponseField>

## Errors

| Code                  | HTTP | Meaning                                             |
| --------------------- | ---- | --------------------------------------------------- |
| `VENUE_NOT_FOUND`     | 404  | One or more venues are unknown                      |
| `INVALID_DATE_RANGE`  | 400  | `from`/`to` missing, inverted, or > 31 days span    |
| `TREATMENT_NOT_FOUND` | 404  | `treatment_id` does not belong to the venue         |

## Example

<RequestExample>

```bash curl
curl "https://api-sandbox.lymfea.com/v1/availability?venue_id=hot_lutetia_paris&treatment_id=trt_swedish_60&from=2026-05-12&to=2026-05-14" \
  -H "Authorization: Bearer $LYMFEA_API_KEY"
```

```javascript Node.js
const params = new URLSearchParams({
  venue_id: "hot_lutetia_paris",
  treatment_id: "trt_swedish_60",
  from: "2026-05-12",
  to: "2026-05-14",
  guests: "1",
});
const res = await fetch(
  `https://api-sandbox.lymfea.com/v1/availability?${params}`,
  { headers: { Authorization: `Bearer ${process.env.LYMFEA_API_KEY}` } }
);
const { data } = await res.json();
```

</RequestExample>
