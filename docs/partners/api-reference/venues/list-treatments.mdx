---
title: "List treatments"
api: "GET /v1/venues/{venue_id}/treatments"
description: "List the active treatments of a venue."
---

Returns the treatment catalogue of a venue, with each treatment's bookable
variants. Requires the `treatments:read` scope.

This endpoint takes **no query parameters** — it returns every active
treatment for the venue, ordered by `sort_order` then `name`.

## Path parameters

<ParamField path="venue_id" type="string" required>
  The venue id.
</ParamField>

## Response

<ResponseField name="data" type="Treatment[]">
  Array of [Treatment](/concepts/treatments) objects, each with its
  `variants` array.
</ResponseField>

## Errors

| HTTP | Body                              | Meaning                              |
| ---- | --------------------------------- | ------------------------------------ |
| 403  | `{ "error": "Forbidden" }`        | Your key is scoped to another venue  |

A venue with no active treatments returns `{ "data": [] }`.

## Example

<RequestExample>

```bash curl
curl https://api.saoma.io/v1/venues/8f1a9c3b-8d4e-4f21-a1b2-c3d4e5f60718/treatments \
  -H "Authorization: Bearer $EIA_API_KEY"
```

```javascript Node.js
const res = await fetch(
  "https://api.saoma.io/v1/venues/8f1a9c3b-8d4e-4f21-a1b2-c3d4e5f60718/treatments",
  { headers: { Authorization: `Bearer ${process.env.EIA_API_KEY}` } }
);
const { data } = await res.json();
```

</RequestExample>
