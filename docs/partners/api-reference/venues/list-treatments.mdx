---
title: "List treatments"
api: "GET /v1/venues/{venue_id}/treatments"
description: "List all bookable treatments of a venue."
---

Returns the treatment catalogue of a venue. Use this endpoint when you only
need the catalogue (e.g. to refresh it on a schedule) without re-fetching
venue metadata.

## Path parameters

<ParamField path="venue_id" type="string" required>
  The venue id or slug.
</ParamField>

## Query parameters

<ParamField query="category" type="string">
  Keep only treatments in this category.
</ParamField>

<ParamField query="service_for" type="'women' | 'men' | 'all' | 'couple'">
  Keep only treatments for this audience.
</ParamField>

<ParamField query="include_addons" type="boolean" default="false">
  Include treatments flagged as add-ons (not bookable alone). By default they
  are filtered out.
</ParamField>

## Response

<ResponseField name="data" type="Treatment[]">
  Array of [Treatment](/concepts/treatments) objects (with variants).
</ResponseField>

## Example

<RequestExample>

```bash curl
curl "https://api-sandbox.lymfea.com/v1/venues/hot_lutetia_paris/treatments?category=massage" \
  -H "Authorization: Bearer $LYMFEA_API_KEY"
```
