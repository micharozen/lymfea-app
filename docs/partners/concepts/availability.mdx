Availability is computed in real-time from the venue's schedule, staff roster
and existing bookings. You can query it for a whole venue
([Get venue availability](/api-reference/availability/get-availability)) or for
a single treatment
([Get treatment availability](/api-reference/availability/get-treatment-availability)).

## Slots

A **slot** is a bookable start time at a venue. Each slot carries:

- `time` — start time in the venue's **local time**, `HH:MM`, aligned on the
  venue's `slot_interval` (typically every 30 minutes).
- `out_of_hours` — `true` when the slot starts outside the posted opening hours;
  booking it applies the venue's `out_of_hours_surcharge_percent`.
- `available_capacity` — how many bookings can still start at this slot, i.e.
  `min(free rooms, free qualified therapists)`. A venue with several treatment
  rooms can take several simultaneous bookings on the same slot.

```json
{
  "date": "2026-06-20",
  "out_of_hours_surcharge_percent": 20,
  "slots": [
    { "time": "08:00", "out_of_hours": true,  "available_capacity": 1 },
    { "time": "09:00", "out_of_hours": false, "available_capacity": 3 }
  ]
}
```

Use the `timezone` field on the parent [Venue](/concepts/venues) to convert a
slot to an absolute instant if you need one.

## How slots are computed

A start time is returned as available only if **all** of the following are true:

1. The venue is deployed on that date (recurring schedule or one-time opening).
2. The slot falls within the venue's daily `opening_time` … `closing_time`
   (widened when the venue allows out-of-hours bookings).
3. The slot respects the treatment's `lead_time` and the venue's minimum booking
   notice from "now".
4. A **treatment room** with sufficient capacity is free for the full duration,
   including the venue's `room_turnover_buffer_minutes`.
5. At least one **therapist** is on shift, qualified for the treatment (skills),
   not already booked, and not blocked by the `inter_venue_buffer_minutes`
   travel buffer.
6. The slot does not fall inside a **blocked time range** configured by the
   venue (lunch break, maintenance, etc.).

When you query for a whole venue (no treatment), the slot duration defaults to
the venue's `slot_interval` and the therapist skill filter is not applied. When
you query for a treatment, its duration (or the selected variant's) and required
skills are used — so treatment availability is always a subset of the venue's.

<Note>
  Because availability depends on live data, a slot returned here may be taken by
  another customer before a booking is made. Always re-check at booking time and
  handle conflicts gracefully.
</Note>

## Querying a date range

Pass `from` and `to` (up to 62 days) instead of `date` to get back the list of
**days** that have at least one bookable slot — ideal for painting a month
calendar before drilling into a specific day.

```json
{ "from": "2026-06-20", "to": "2026-06-30", "days": ["2026-06-20", "2026-06-24"] }
```
