A **Venue** is the physical place where treatments are delivered. There are
two types of venues on Saoma (`venue_type`):

- `hotel` — a spa inside a hotel. Guests can charge their booking to their
  room (the room-number field applies).
- `spa` — an independent day spa. No room charge.

<Note>
  The Venue object comes in two shapes. The **list shape** (returned by
  `GET /v1/venues`) carries marketing/contact fields. The **detail shape**
  (returned by `GET /v1/venues/{id}`) carries scheduling and booking-policy
  fields. The two shapes do not currently expose the exact same set of fields.
</Note>

## List shape — `GET /v1/venues`

<ResponseField name="id" type="string">
  Stable, opaque identifier (UUID) for the venue.
</ResponseField>

<ResponseField name="slug" type="string">
  Human-readable identifier, unique within your organization (kebab-case,
  e.g. `akasha-lutetia`). Stable: it does **not** change when the venue is
  renamed. Any endpoint that accepts `{venue_id}` in the URL also accepts the
  slug — use whichever you prefer when integrating.
</ResponseField>

<ResponseField name="name" type="string">
  Display name in French.
</ResponseField>

<ResponseField name="name_en" type="string | null">
  Display name in English.
</ResponseField>

<ResponseField name="description" type="string | null">
  Long-form description in French.
</ResponseField>

<ResponseField name="description_en" type="string | null">
  Long-form description in English.
</ResponseField>

<ResponseField name="address" type="string | null">
  Street address.
</ResponseField>

<ResponseField name="postal_code" type="string | null">
  Postal or ZIP code.
</ResponseField>

<ResponseField name="city" type="string | null">
  City.
</ResponseField>

<ResponseField name="country" type="string | null">
  ISO 3166-1 alpha-2 country code (e.g. `FR`).
</ResponseField>

<ResponseField name="currency" type="string">
  ISO 4217 currency code for pricing (e.g. `EUR`).
</ResponseField>

<ResponseField name="venue_type" type="'hotel' | 'spa'">
  Whether the venue is a hotel spa or a standalone day spa.
</ResponseField>

<ResponseField name="timezone" type="string | null">
  IANA timezone (e.g. `Europe/Paris`).
</ResponseField>

<ResponseField name="image" type="string | null">
  Primary image URL.
</ResponseField>

<ResponseField name="cover_image" type="string | null">
  Cover image URL, suitable for a hero banner.
</ResponseField>

## Detail shape — `GET /v1/venues/{id}`

Includes `id`, `name`, `image`, `cover_image`, `city`, `country`, `currency`,
`venue_type`, `description` and `landing_subtitle`, plus the scheduling and
policy fields below.

<ResponseField name="status" type="string">
  Lifecycle status (e.g. `active`).
</ResponseField>

<ResponseField name="vat" type="number">
  VAT rate applied to prices, as a percentage (e.g. `20`).
</ResponseField>

<ResponseField name="opening_time" type="string">
  Daily opening time in `HH:mm:ss`, in the venue's local timezone.
</ResponseField>

<ResponseField name="closing_time" type="string">
  Daily closing time in `HH:mm:ss`, in the venue's local timezone.
</ResponseField>

<ResponseField name="slot_interval" type="integer">
  Granularity of bookable slots, in minutes (typically `30`).
</ResponseField>

<ResponseField name="schedule_type" type="string">
  How the venue's availability is defined (e.g. `recurring`).
</ResponseField>

<ResponseField name="days_of_week" type="integer[]">
  Days the venue operates, `0` (Sunday) … `6` (Saturday).
</ResponseField>

<ResponseField name="recurrence_interval" type="integer | null">
  Recurrence interval for the schedule.
</ResponseField>

<ResponseField name="recurring_start_date" type="string (ISO date) | null">
  First date the recurring schedule applies.
</ResponseField>

<ResponseField name="recurring_end_date" type="string (ISO date) | null">
  Last date the recurring schedule applies, or `null` for open-ended.
</ResponseField>

<ResponseField name="offert" type="boolean">
  When `true`, the venue currently offers complimentary treatments.
</ResponseField>

<ResponseField name="booking_hold_enabled" type="boolean">
  Whether slots are temporarily held while a customer completes payment.
</ResponseField>

<ResponseField name="booking_hold_duration_minutes" type="integer">
  How long a slot is held when `booking_hold_enabled` is `true`.
</ResponseField>

<ResponseField name="allow_out_of_hours_booking" type="boolean">
  Whether bookings outside opening hours are accepted.
</ResponseField>

<ResponseField name="out_of_hours_surcharge_percent" type="number">
  Surcharge applied to out-of-hours bookings, as a percentage.
</ResponseField>

<Note>
  Only venues with `status = 'active'` (and in scope for your API key) are
  returned by `GET /v1/venues`. Inactive or archived venues are filtered out
  automatically.
</Note>
